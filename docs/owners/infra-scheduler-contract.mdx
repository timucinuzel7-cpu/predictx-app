---
title: Scheduler Contract
description: Canonical `/api/sync/*` schedule and request contract for non-Vercel scheduler setups.
---

<Callout type="info">
  If you run Supabase mode, `npm run db:push` configures `pg_cron` with this same schedule.
</Callout>

## Request contract

- Method: `GET`
- Header: `Authorization: Bearer <CRON_SECRET>`
- Base URL: `SITE_URL`
- Endpoints:
  - `/api/sync/events`
  - `/api/sync/resolution`
  - `/api/sync/translations`
  - `/api/sync/volume`

Example:

```bash
curl -sS -X GET \
  -H "Authorization: Bearer ${CRON_SECRET}" \
  "${SITE_URL}/api/sync/events"
```

---

## Canonical schedules

| Endpoint | Cron schedule | Notes |
| --- | --- | --- |
| `/api/sync/events` | `1-59/5 * * * *` | Every 5 min (offset by 1 min) |
| `/api/sync/resolution` | `3-59/5 * * * *` | Every 5 min (offset by 3 min) |
| `/api/sync/translations` | `*/10 * * * *` | Every 10 min |
| `/api/sync/volume` | `14,44 * * * *` | Every 30 min (minute 14 and 44) |

Use UTC unless your team has a strict local-time requirement.

---

## Runtime expectations

- `200`: accepted/completed
- `401`: missing/invalid `CRON_SECRET`
- `409`: sync already running (expected occasionally for locked routes)
- `500`: runtime failure (investigate logs)

<Callout type="warning">
  Do not run duplicate schedulers simultaneously (`pg_cron` + external scheduler), unless you intentionally accept duplicate calls.
</Callout>
