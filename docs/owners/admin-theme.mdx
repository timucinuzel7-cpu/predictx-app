## What this section controls

| Control | Purpose |
| --- | --- |
| **Preset** | Selects the initial visual palette. |
| **Corner roundness** | Sets the global radius style used by cards, inputs, and buttons. |
| **Theme tokens** | Light/dark overrides for core, brand, outcome, and chart colors. |
| **Preview cards** | Live preview for light and dark themes before saving. |

---

## Recommended workflow

1. Choose a preset close to your brand direction.
2. Set corner roundness to match your product tone.
3. Tune token groups in this order: core, brand, outcomes, chart.
4. Validate contrast in both Light and Dark previews.
5. Save and quickly review market cards, order actions, and docs pages.

---

## Common mistakes to avoid

- Overriding too many tokens before locking a preset baseline.
- Reducing contrast on muted text and borders too aggressively.
- Making YES/NO colors too similar for fast trading decisions.
