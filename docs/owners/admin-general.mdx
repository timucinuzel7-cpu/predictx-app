## What this section controls

| Block | Main controls |
| --- | --- |
| **Brand identity** | Company name, company description, logo upload, PWA install icons. |
| **Community and analytics** | Google Analytics ID, Discord community link, support URL. |
| **OpenRouter integration** | API key, preferred model, model refresh. |
| **LI.FI integration** | Integrator name and API key (optional). |
| **Market and fee settings** | Fee recipient wallet and allowed market creator wallets. |

---

## Recommended first-pass setup

1. Set company name/description and upload logo.
2. Configure support link and Discord link.
3. Set fee recipient wallet (Polygon address).
4. Add allowed market creators if your fork should filter creator scope.
5. Configure OpenRouter only if you plan to use Market Context or automatic translations.

---

## Operational notes

- OpenRouter model selection depends on a valid OpenRouter API key.
- Allowed market creators can be entered one per line (or comma-separated).
- Keep support link up to date with your real escalation channel.
