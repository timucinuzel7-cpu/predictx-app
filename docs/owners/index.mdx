## High-level flow

<Steps>
  <Step>
    ### Fork the repository

    Create the fork on GitHub and keep the project name short so it is easy to locate from Vercel.
  </Step>
  <Step>
    ### Prepare environment variables

    Download `.env.example`, fill in required values, and keep this file ready for import during Vercel project setup.
  </Step>
  <Step>
    ### Import into Vercel and run first deploy

    From **Vercel -> Add New -> Project**, connect GitHub, import your fork, then use **Import .env** before deploying.
  </Step>
  <Step>
    ### Provision Supabase

    Use **Storage -> Supabase** to create and connect a database to the project.
  </Step>
  <Step>
    ### Redeploy after database connection

    Redeploy the latest build so the app starts with the database integration active.
  </Step>
  <Step>
    ### Enable GitHub Actions sync

    In your fork, allow GitHub Actions so upstream sync workflows can run.
  </Step>
  <Step>
    ### Finish setup in Admin

    Log in with an admin wallet and complete the first pass in [**Admin -> General**](/docs/owners/admin-general).
  </Step>
  <Step>
    ### Add optional integrations

    Configure OpenRouter and Market Context only if your fork needs AI-generated context/translations.
  </Step>
</Steps>

## What you'll find here

- **Environment reference:** only the env vars that affect production launches.
- **Network proxy setup:** how to route Kuest infrastructure through your own domain.
- **Infra section (non-Vercel):** deployment targets, storage-mode decisioning, and scheduler contract.
- **Admin guides:** one page for each admin section: General, Theme, Locales, Categories, Market Context, Affiliate, Events, and Users.
