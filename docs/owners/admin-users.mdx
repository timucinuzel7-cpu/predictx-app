## What this section shows

| Column | Description |
| --- | --- |
| **User** | Profile identity plus admin badge when applicable. |
| **Email** | Mail contact action (when email is available). |
| **Referral** | Upstream referral attribution when present. |
| **Created** | Account creation timestamp. |

---

## Typical use cases

1. Investigate support tickets by username/email.
2. Validate whether a specific account has admin privileges.
3. Audit referral attribution in dispute scenarios.
4. Monitor user growth over time via created dates.

---

## Notes

- This page is optimized for discovery and review; operational role assignment is managed by `ADMIN_WALLETS` configuration.
